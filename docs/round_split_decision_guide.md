# Round Split Decision Guide

这份文档定义：什么时候默认建议拆轮，什么时候仍可压成单轮，但必须明确边界。

目标不是把任务拆得越碎越好，而是避免把多个主闭环硬塞成一轮。

## 1. 默认原则

只要当前输入无法稳定收成一个主闭环，就默认：

- `Prompt Gate = REWRITE_REQUIRED`
- 并建议拆轮

拆轮不是形式动作，而是为了保证：

- 本轮目标可完成
- 本轮验收可判断
- 本轮测试可执行
- 本轮回传可收口

## 2. 默认建议拆轮的情形

### 1. 多个核心能力主题同时出现

典型信号：

- 例如“导出 EPUB + 续写半章 + 风格引导”
- 每个主题都足以单独形成一轮主目标

默认动作：

- `REWRITE_REQUIRED`
- 建议拆轮

### 2. 后端逻辑 + API + UI 同时为新实现

典型信号：

- 同一轮既要设计数据逻辑，又要接 API，又要补页面交互
- 且这些都不是小接线，而是新的主实现

默认动作：

- `REWRITE_REQUIRED`
- 建议按“最先形成可验收闭环的主路径”拆轮

### 3. 新能力实现 + 体验优化 / 风格优化混合

典型信号：

- 一边要求“先把功能做出来”
- 一边要求“顺便把体验、文案、风格、引导也一起优化”

默认动作：

- `REWRITE_REQUIRED`
- 先保留主功能闭环
- 优化项通常顺延到 `polish` 轮

### 4. 依赖接入 + 功能实现 + 界面改造混合

典型信号：

- 同时接第三方依赖、补业务能力、改 CLI/API/Web UI
- 且任一块失败都会拖垮整轮验收

默认动作：

- `REWRITE_REQUIRED`
- 优先拆出“解依赖 / 解阻轮”或“主路径闭环轮”

### 5. 同一轮验收项无法收成一个主闭环

典型信号：

- 验收项之间没有单一主路径
- 做完之后无法用一句话回答“这轮到底完成了什么”

默认动作：

- `REWRITE_REQUIRED`
- 先重写目标，再决定是否拆轮

## 3. 哪些情况下默认 REWRITE_REQUIRED 且建议拆轮

满足下列任两项以上时，默认：

- `Prompt Gate = REWRITE_REQUIRED`
- 建议拆轮

判断条件：

- 出现两个以上独立核心能力主题
- 同时包含新能力实现与优化类目标
- 同时包含依赖接入与业务实现
- 同时包含 CLI / API / Web UI 多入口改造
- 依赖不确定项足以影响验收口径
- 当前“本轮不做”很难写清
- 当前验收标准需要分成两个以上主路径才能表达

## 4. 哪些情况下可压成单轮，但必须写清降级边界

下列情况可以不拆轮，但必须显式写清边界与降级：

### 1. 主要只有一个主闭环，其他内容只是必要接线

例子：

- 一个现有页面从 mock 数据改为真实 API
- 同时补最基础 loading / error / empty state

要求：

- 必须写明主闭环是什么
- 必须写明哪些接线只是为这个闭环服务

### 2. 依赖接入只是主路径的必要组成，而不是独立目标

例子：

- 为了让导出按钮可用，需要补一个已定义接口接线

要求：

- 必须写明依赖满足时怎么做
- 依赖不满足时降到哪里
- 哪些情况直接阻塞

### 3. UI 改动只是主闭环的可见结果，不是第二主目标

例子：

- 为了让新功能可用，必须补一个最基础结果展示页

要求：

- 不得顺手扩大成整套体验重做
- `不做` 里要显式排除风格优化、全面改版等内容

## 5. 推荐拆轮方式

如果需要拆轮，优先按下面方式拆：

### 1. 先拆“最小可验收主闭环”

先保留：

- 最能推进项目闭环的一件事

后移：

- 体验提升
- 风格引导
- 非核心入口
- 次要模块

### 2. 先拆“解阻轮”和“正式实现轮”

适用于：

- 当前主要问题是依赖、环境、权限、接口契约不稳

先做：

- `unblock`

再做：

- `build`

### 3. 先拆“主路径轮”和“扩展路径轮”

适用于：

- 同时出现 CLI / API / Web UI

先做：

- 最关键入口的主路径闭环

后做：

- 第二入口
- 第三入口

### 4. 先拆“功能轮”和“polish 轮”

适用于：

- 功能实现和风格 / 体验优化混在一起

先做：

- 能过验收的功能闭环

后做：

- 文案、引导、风格、交互细节优化

## 6. 拆轮建议输出要求

一旦建议拆轮，Gate 结果里至少要写清：

- 为什么当前输入不是单轮最小闭环
- 建议拆成几轮
- 每轮各自的主目标
- 哪一轮最适合先做
- 当前轮明确不做什么

如果这些写不出来，说明还没有真正完成拆轮判断。
