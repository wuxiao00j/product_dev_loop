# 示例：执行器说已完成，但测试说明里仍需重点验证

## 当前回执关键信息

- 执行器声称：已接入待办保存链路，刷新后应仍保留新增结果
- 已知风险：没有提供足够测试证据
- 已知限制：删除功能仍未接线

## 生成后的用户可读测试说明示例

```md
# Todo Lite Round 04 测试说明

## 本轮重点验证项

- 新增一条待办后，刷新页面结果是否仍保留
- 页面是否不再依赖明显假数据

## 需重点验证的声称完成项

- 执行器声称“刷新后结果会保留”，本轮请重点验证这一点，不要默认视为已闭环

## 测试入口

- 页面：`/todos`

## 测试前提

- 开发环境已启动
- 当前可直接访问 `/todos`

## 操作步骤

1. 打开 `/todos`
2. 输入标题 `买牛奶`
3. 点击提交
4. 确认页面出现新增结果
5. 刷新页面
6. 再次确认 `买牛奶` 是否仍存在

## 测试数据

- 标题：`买牛奶`

## 预期结果

- 提交后立即看到新增结果
- 刷新后仍能看到同一条待办

## 失败判定

- 提交后没有新增结果
- 刷新后新增结果丢失
- 页面依然显示明显示例假数据，无法判断是否真实新增

## 已知限制 / 暂不覆盖范围

- 删除功能仍未接线，不在本轮验证范围内
```

## 这个示例强调什么

- 即使执行器说“已完成”，测试说明里也可以明确写“仍需重点验证”
- 用户读到的是可执行步骤，不是代码描述
